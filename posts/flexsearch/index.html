基于 FlexSearch 的 Hugo 集成案例

2026-05-02

1. 配置 Hugo 生成搜索索引 (JSON)

首先，你需要让 Hugo 在构建时输出一个 index.json 文件。

修改你的 hugo.yaml (或 config.toml)：

YAML

outputs:
  home:
    - HTML
    - JSON # 开启首页的 JSON 输出

然后创建模板文件 layouts/_default/index.json：

注意：这里只索引标题、标签和摘要，以平衡搜索准确度和文件体积。

JSON

[
  {{- range $index, $page := .Site.RegularPages -}}
  {{- if $index -}},{{- end -}}
  {
    "title": {{ $page.Title | jsonify }},
    "permalink": {{ $page.Permalink | jsonify }},
    "tags": {{ $page.Params.tags | jsonify }},
    "content": {{ $page.Plain | jsonify }}
  }
  {{- end -}}
]

2. 创建搜索页面模板

创建 layouts/_default/search.html。这里我们引入 FlexSearch 并编写中文优化的逻辑。

HTML

{{ define "main" }}
<div class="search-wrap">
    <input type="text" id="search-input" placeholder="输入关键词，支持中文精确搜索..." />
    <div id="search-results"></div>
</div>

<!-- 引入 FlexSearch -->
<script src="https://cdn.jsdelivr.net/gh/nextapps-be/flexsearch@0.7.31/dist/flexsearch.bundle.js"></script>

<script>
    // 1. 初始化 FlexSearch 索引 (中文优化配置)
    const index = new FlexSearch.Document({
        tokenize: "forward", // 针对中文建议使用 forward 或自定义
        cache: true,
        document: {
            id: "id",
            index: ["title", "content", "tags"], // 索引这些字段
            store: ["title", "permalink"] // 结果中直接返回这些字段
        }
    });

    const searchInput = document.getElementById('search-input');
    const resultsContainer = document.getElementById('search-results');

    // 2. 加载 Hugo 生成的 JSON 并建立索引
    fetch('/index.json')
        .then(response => response.json())
        .then(data => {
            data.forEach((doc, i) => {
                index.add({
                    id: i,
                    title: doc.title,
                    content: doc.content,
                    tags: doc.tags ? doc.tags.join(' ') : ''
                });
                // 将原始数据存入内存以便渲染结果
                window.searchData = data;
            });
            console.log("搜索索引已就绪");
        });

    // 3. 监听输入进行搜索
    searchInput.addEventListener('input', function() {
        const query = this.value;
        resultsContainer.innerHTML = '';

        if (query.length < 2) return;

        // 执行多字段搜索
        const results = index.search(query, { limit: 10, enrich: true });

        if (results.length > 0) {
            // 合并来自 title, content, tags 字段的搜索结果并去重
            const allIndices = new Set();
            results.forEach(res => {
                res.result.forEach(r => allIndices.add(r.id));
            });

            allIndices.forEach(id => {
                const item = window.searchData[id];
                const div = document.createElement('div');
                div.className = 'search-result-item';
                div.innerHTML = `<h4><a href="${item.permalink}">${item.title}</a></h4>`;
                resultsContainer.appendChild(div);
            });
        } else {
            resultsContainer.innerHTML = '<p>未找到匹配结果</p>';
        }
    });
</script>

<style>
    /* 基础样式，你可以根据你的暗黑模式进行微调 */
    .search-wrap { max-width: 700px; margin: 40px auto; }
    #search-input { width: 100%; padding: 12px; font-size: 18px; border: 2px solid #ddd; border-radius: 8px; }
    .search-result-item { padding: 15px 0; border-bottom: 1px solid #eee; }
    .search-result-item h4 { margin: 0; }
    .search-result-item a { color: #007bff; text-decoration: none; }
</style>
{{ end }}

3. 为什么这个方案比 Pagefind 准确？

  1. Tokenize 策略：在上面的配置中，我们使用了 tokenize: "forward"。对于中文来说，这意味着它会更智能地处理词头匹配。

  2. 多字段权重：你可以通过 FlexSearch 分别搜索 title 和 content，并在展示时优先合并 title 的结果。

  3. 完全掌控：Pagefind 是黑盒，而 FlexSearch 允许你通过 split 正则表达式自定义分词逻辑。如果觉得还不准，可以将 tokenize 改为：

    JavaScript

    tokenize: function(str){
    return str.replace(/[\x00-\x7F]/g, " ").split(/\s+/);
}

    这种方式会将中文字符拆解，实现极高密度的倒排索引，几乎能搜到任何包含的字。

4. 配合你的暗黑模式

你之前提到过 CSS 的暗黑模式调试。对于 FlexSearch，由于 UI 是你自己写的，你可以完全控制：

CSS

@media (prefers-color-scheme: dark) {
    #search-input {
        background: #2d2d2d;
        color: #fff;
        border-color: #444;
    }
    .search-result-item a {
        color: #8ab4f8;
    }
}

总结

  • Pagefind：适合文章极多（上千篇）、追求极致部署速度、对中文准确度要求适中的场景。
  • FlexSearch：适合对搜索结果有“洁癖”、希望中文匹配非常精准的博客。

由于你之前已经手动处理过 JSON 索引，这个方案对你来说应该更亲切。你现在可以尝试运行 hugo 生成项目，然后通过 index.json 看看 FlexSearch 的反馈。如果有字段匹配不到，我们可以调整 index.json 的字段结构。
