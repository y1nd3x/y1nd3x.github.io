修复Windows下换行符CRLF问题

2026-05-05

在 Windows 上开发使用 Git 时，经常会遇到 CRLF 与 LF 的冲突报错问题，解决办法如下：

Quick Fix

git config --global core.autocrlf input

提交时转 LF，拉取时不转换，并且配置保存在 .gitconfig 文件中便于备份恢复。

Full Fix

在 Git Bash for Windows 中执行如下命令转换所有markdown文档：

find . -type f \( -iname "*.md" -o -iname "*.markdown" \) -exec dos2unix {} +

转换完成后，使用 file xxx.md 命令查看文件属性，输出包含 with CRLF line terminators表示还没改，输出仅显示 UTF-8 Unicode text表示已经是 LF 格式了。你可以运行以下命令来确认是否还有残留的 CRLF 文件。

find . -type f \( -name "*.md" -o -name "*.markdown" \) -exec file {} + | grep "CRLF"

*IDE Config

Ensure that all tools use LF line endings and UTF-8 (without BOM) encoding by default when creating and saving files.

1. VS Code

  • Shortcut: Ctrl + , (Comma) to open Settings.
  • Line Endings: Search for files.eol and change the value from auto to \n.
  • Encoding: Search for files.encoding and ensure it is set to utf8.
  • Auto Guess Encoding: Search for files.autoGuessEncoding and check the box for true (This helps prevent garbled text when opening older files).

2. Typora

  • Path: File -> Preferences -> Editor.
  • Line Endings: Change Default Line Ending to LF.
  • Encoding: Ensure Default Encoding is set to UTF-8 (Note: In newer versions, this is the hardcoded default and the explicit option may be hidden).
  • Status Bar: View -> Status Bar. (Once enabled, you can see and toggle between LF and CRLF in the bottom right corner of the window).

3. Notepad++

  • Path: Settings -> Preferences -> New Document.
  • Line Endings: Under the Format section, select Unix (LF).
  • Encoding: Select UTF-8 (Ensure “Apply to opened ANSI files” is unchecked if you want to keep it strictly UTF-8).
  • Convert Existing Files: Edit -> EOL Conversion -> Unix (LF).
