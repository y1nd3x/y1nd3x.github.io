使用Nginx搭建简易本地Web服务器

2026-05-01

参数配置

在服务器上安装 Nginx 后，编辑配置文件（通常在 /etc/nginx/conf.d/hugo.conf 或 /etc/nginx/sites-available/default）

server {
    listen       80;
    server_name  192.168.1.100; # 你的服务器内网 IP

    location / {
        root   /var/www/hugo;    # hugo->public files
        index  index.html index.htm;
        
        # 即使是静态站点，建议加上这个，防止 Hugo 的某些路径出现 404
        try_files $uri $uri/ =404;
    }

    # 可选：开启 Gzip 压缩，让网页加载更快
    gzip on;
    gzip_types text/plain text/css application/javascript;
}

部署步骤（最快流程）

sudo mkdir -p /var/www/hugo								##创建服务目录
scp -r ./public/* user@192.168.1.100:/var/www/hugo		##拷贝网站文件
sudo chown -R www-data:www-data /var/www/hugo			##确保 Nginx 权限正确
sudo nginx -t											##检查配置语法是否正确  
sudo systemctl restart nginx							##重启服务

更新推送

在你的 Hugo 项目根目录下，执行以下命令即可将生成的静态文件推送到服务器

PowerShell

scp -r ./public/* your_user@192.168.1.100:/var/www/hugo

-r		#递归拷贝整个文件夹内容
./public/*	#只拷贝 public 文件夹内部的文件，而不拷贝 public 这个文件夹本身
your_user	#你在服务器上的用户名（如 root 或其他用户）

为了不用每次都敲长命令，建议在项目根目录下创建一个名为 deploy.bat 的文件，内容如下

@echo off
echo [1/2] 正在清理并重新生成静态站点...
hugo --cleanDestinationDir

echo [2/2] 正在通过 SCP 推送到服务器...
:: 请将下面的 root 修改为你服务器的用户名
scp -r ./public/* root@192.168.1.100:/var/www/hugo

echo ---------------------------------------
echo 部署完成！
pause

配置 SSH Key（免密登录）

  1. ​ 在 Windows 终端执行

    ssh-keygen -t rsa（一路回车即可）

  2. ​ 将公钥发送到服务器（最简单的方法）

type $env:USERPROFILE\.ssh\id_rsa.pub | ssh your_user@192.168.1.100 "cat >> ~/.ssh/authorized_keys"
